Pet Smart Door mobile application.

Smart door is a company that provides connected doors for pets.
The client orders the door, installs it, and then he can control the door using the mobile app.

1. The challenge : More than a pet door

To stand out from the competition, the goal is to go above the basic connected door, and to provide the owner real time data about his animals. The user can control the door but also see at any times the current location of the animals. And an activity report is available to see how much time was spent outside per day / week or month.

App main functions

  • - Lock or unlock the door at any times.
  • - Register the animals with their ID.
  • - Schedule a time restriction.
  • - See the current location and record the animal activity.
  • - Control several doors.

2. Information architecture

The initial onboarding process had several steps. The user had to enter his informations, connect the door, manage the door settings and finally register their pet ID.

3. User test : paper prototype revealed that the onboarding process had too many steps.

Users wanted to be able to quickly synchronize the door, test it, and only later on be able to identify his pets id and set up a door schedule. For most users, one of the most important functions of the app is to be able to quickly see where the animals are at all times.

4. Sketches

Rapid sketch iterations were used to make decision about content organisation. The main page needed to have two interfaces designed. One for the first use, with buttons inviting the user to identify the animals and set a time restriction. And an other one once this steps where completed.

5. Wireframess

I set up a lo-fi clickable prototype using figma to help aid the customer conversations and also provide usability insights.

6. Low fidelity prototype user test

To test the prototype, I used maze. I created 3 missions for the user :

  • - Create an account
  • - Register an animal
  • - Add a time restriction to the door

Results and solution

Time restriction issue : It appeared that it was not clear that when the door was locked, the animals could still enter the house but not go outside.
Solution : I added a banner on top of the time restriction page, with the message : "Lock time : Your animals can always enter through the door even after the lock time." The banner stays until the user dismisses it.

7. High fidelity prototypes

Account creation

The first step for the user is to create an account. Then he is invited to connect the door to the app. Once succesfully connected, the user is directed on the main page where he can register his animal and create a schedule.

Lock / Unlock the door

On the main page the user can lock and unlock the door. The color code and the icon help to see quickly the sate of the door.

Register your animals

Two buttons ont the main page invite the user to register the animals and add a schedule. By registering the animals microship, the door will only work for them, preventing foreign animals to enter the home.

Timeline of activity

The user can see the animals activity by day, week and month. He can see how much time was spent outside. And at what hour the pet was last seen into the house. The design was created following material design tabs guidelines.

Add a schedule
